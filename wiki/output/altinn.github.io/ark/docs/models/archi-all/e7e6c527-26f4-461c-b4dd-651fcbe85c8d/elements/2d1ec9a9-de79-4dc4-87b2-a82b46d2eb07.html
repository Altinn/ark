Issues tjeneste ()
Som for all annen programvareutvikling, så vil hver 3.0-tjeneste typisk ha en egen backlog med bugs, forbedringer, fremtidige features, etc. som tjenesteutviklerne vil jobbe mot. Dette er en sentral del av Application Lifecycle Management. - opprette og lukke issues i 3.0-tjenesten - tid for opprettelse - hvem opprettet/lukket - filtrering av issues på status - legge til etikett - filtrering av issues pr. etikett - knytt issues til milepæl - filtrer issues pr milepæl - tilordne saker til bruker - filtrere saker pr bruker - redigere markdown - diskusjoner - legge til filer til issues - notifikasjon når saker endres - søk etter issues - listing av issues
