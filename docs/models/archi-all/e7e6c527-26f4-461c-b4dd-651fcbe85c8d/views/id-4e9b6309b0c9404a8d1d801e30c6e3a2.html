Behøver Dialogporten egen funksjonalitet for varlsing? Ellers kan vel Meldingskomponenten selv sende varsel? Behøves grenseoppgang mellom produktene? Ser rart ut at API for avsendere ligger hos Meldingskomponenten, mens API for mottakere ligger hos Dialogporten. Forklaring? * Bjørns utkast 1 viser ikke sending og mottak av meldinger via GUI (felles arbeidsflate), noe som gjør det litt vanseklig å forstå. * Presisering: Meldingmottaker som mottar via API kan rute videre internt til sluttbruker i eget sluttbrukersystem (SBS) * Figuren bør kanskje la skillet mellom Altinn felles arbeidsflate og SBS komme tydeligere fram? Skill mellom * Samme betraktning om SBS vs. Altinn felles arbeidsflate gjelder vel for avsender? * Gjerne forklar sammenhengen tjenesteeier vs. tjenestetilbyder? * Finnes noen eier av produktet Altinn storage? Dokumentasjon? Terje Holene? * Få med noe om adressering m.m.
