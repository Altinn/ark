2.4 Tjenester 3.0 Runtime ()
“Altinn Runtime” er løsningen som skal kjøre 3.0-tjenestene som er utviklet med Designer og som lagres i Repositories. Eksakt samme runtime vil kjøre ved enhetstest i Designer som i Altinns test- og produksjonsmiljøer. Denne vil kjøre i parallell med dagens ServiceEngine og Archive, og det å integrere den nye løsningen i den gamle på en hensiktsmessig måte, men uten for mange kompromisser, blir en hovedutfordring og noe som vil være viktig å ha fokus på. Hypotesen er at “Runtime” bør deles i to: API’er som behandler brukerdata vil kjøre i dagens Altinn-infrastruktur (f.eks. logikk, validering, etc) Front-end og API’er som ikke inneholder brukerdata vil serves fra skyen (f.eks. kodelister, web komponenter, statiske assets, etc) Dette gjør at vi kan utnytte en ekte sky for Designer og Repositories, samtidig som brukernes data ikke behøver flyte via en sky i utlandet, men kun via brukerens nettleser og API’er i norge.
